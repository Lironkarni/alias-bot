# בוט אליאס לטלגרם

בוט משחק אליאס לקבוצות טלגרם, כולל מסך משחק (Mini App) פרטי לכל שחקן עם טיימר וניקוד בזמן אמת.

## מבנה הפרויקט

```
server.js        # שרת Express + Socket.io + הפעלת הבוט
bot.js            # פקודות וכפתורים של הבוט בקבוצה
gameManager.js    # כל הלוגיקה של המשחק (לובי, קבוצות, תורות, ניקוד)
telegramAuth.js   # אימות initData שמגיע מה-Mini App
words.js          # רשימת המילים
public/           # מסך המשחק (ה-Mini App עצמו)
```

## שלב 1: יצירת הבוט וה-Mini App ב-BotFather

1. פתחו שיחה עם [@BotFather](https://t.me/BotFather) ושלחו `/newbot`, בחרו שם ו-username.
   שמרו את ה-**token** שתקבלו.
2. שלחו `/newapp`, בחרו את הבוט שיצרתם, ותנו ל-Mini App שם קצר (short name), לדוגמה `play`.
   כשיבקשו כתובת URL - **תחזרו לזה אחרי שלב 2** (אחרי שתדעו את הכתובת הציבורית של השרת).
3. חשוב: בקבוצה, הבוט חייב הרשאות לשלוח הודעות (ואם תרצו שיוכל להוסיף את עצמו, גם privacy mode כבוי דרך `/setprivacy` אם תרצו שיקרא גם הודעות רגילות - לא חובה למשחק הזה כי משתמשים רק בכפתורים ופקודות).

## שלב 2: פריסה (Railway / Render)

1. דחפו את הפרויקט הזה ל-GitHub repo.
2. ב-Railway או Render: New Project → Deploy from GitHub → בחרו את ה-repo.
3. הגדירו משתני סביבה (Environment Variables) לפי `.env.example`:
   - `BOT_TOKEN`
   - `BOT_USERNAME`
   - `MINIAPP_SHORTNAME` (מה שנתתם ל-BotFather, למשל `play`)
   - `PUBLIC_URL` — תקבלו כתובת כזו מהפלטפורמה אחרי הפריסה הראשונה (למשל `https://alias-bot.up.railway.app`)
   - `USE_WEBHOOK=true`
   - `PORT` — בד"כ מוזרק אוטומטית, אין צורך להגדיר ידנית
4. אחרי שיש לכם כתובת `PUBLIC_URL` סופית, חזרו ל-BotFather ל-`/myapps` → בחרו את האפליקציה → **Edit Web App URL** → הכניסו את אותה כתובת (בלי נתיב, רק ה-domain).
5. Redeploy אחרי שהגדרתם את `PUBLIC_URL` הנכון.

## הרצה מקומית (לבדיקות)

```bash
npm install
cp .env.example .env   # ומלאו את הפרטים, השאירו USE_WEBHOOK=false
npm start
```

בפיתוח מקומי (`USE_WEBHOOK=false`) הבוט עובד ב-long polling כך שאין צורך בכתובת ציבורית כדי שהבוט יגיב בקבוצה — אבל כדי לבדוק את מסך ה-Mini App בפועl מתוך טלגרם עדיין צריך כתובת HTTPS ציבורית (אפשר להשתמש ב-[ngrok](https://ngrok.com) זמנית ולעדכן את ה-Web App URL ב-BotFather בהתאם).

## איך זה עובד בפועל

- `/start` בקבוצה פותח לובי עם כפתורי "הצטרף למשחק" ו"התחל משחק" (רק שולח הפקודה יכול ללחוץ להתחיל).
- בהתחלת משחק המשתתפים מתערבבים אקראית ומתחלקים ל-2 קבוצות; סדר השחקנים בכל קבוצה נשמר לאורך המשחק.
- בכל תור הבוט שולח כפתור קישור (`t.me/<bot>/<app>?startapp=<token>`) שמכיל טוקן חד-פעמי הקשור למשתמש הספציפי שאמור להסביר.
- כשמישהו פותח את הקישור, ה-Mini App שולח לשרת את ה-`initData` החתום של טלגרם + הטוקן. השרת מאמת את החתימה (HMAC) ומוודא שה-`user.id` תואם לשחקן הצפוי — כך שגם אם מישהו אחר ילחץ על אותו קישור, הוא יקבל הודעת שגיאה ולא יראה מילים.
- הטיימר של 60 שניות מנוהל בשרת (לא בקליינט), כדי שלא ניתן "לרמות" ע"י רענון הדף.
- ניקוד מצטבר עד 30 נקודות לקבוצה מנצחת.

## הרחבות אפשריות

- שמירת state ב-Redis/SQLite כדי לשרוד ריסטארט שרת.
- מסך "קבוצה שלי" נפרד שמראה גם לשחקנים שלא בתור את הניקוד החי.
- קטגוריות מילים לבחירה מראש (מנהל בוחר קושי/נושא בלובי).
